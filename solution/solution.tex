\section*{做法}

方案数，又不像是组合数学，考虑动态规划

若最后一个格子未涂色，则倒数第二个格子可涂可不涂；若最后一个格子涂色了，则倒数第二个格子一定不能涂色，于是：

\begin{itemize}
    \item 状态设计：$dp[i]$ 代表已考虑了前 $i$ 个格子的方案数
    \item 初始状态：$dp[1] = 2,~dp[2] = 3$
    \item 转移方程：$dp[i] = dp[i-1] + dp[i-2]$
    \item 所求结果：$dp[n]$
\end{itemize}

即斐波那契数列，注意到 $n \leq {10}^{18}$ 很大，需要使用矩阵加速
